# Monks - Transformation Tech - Automated Newsletter

A Next.js website that renders weekly curated newsletters from JSON files generated by your n8n automation.

## What it includes

- Home page highlighting the latest newsletter
- History page listing every newsletter issue
- Dynamic newsletter pages generated at build time
- Ordered sections per issue:
  - Useful Links
  - Maybe Useful Links
  - Discarded Links (collapsed by default)
- Link-level analytics tracking (clicks)
- Thumbs up / thumbs down tracking on each link

## Stack

- Next.js (App Router) + TypeScript
- Static generation from local JSON files
- Vercel-friendly build output

## Local development

```bash
npm install
npm run dev
```

Open http://localhost:3000.

## Build for Vercel

Vercel will run `npm run build` on pushes to `main`. This app generates newsletter routes at build time from files in `data/newsletters`.

```bash
npm run build
npm start
```

## JSON input format

Drop newsletter JSON files into `data/newsletters`.

Example schema:

```json
{
  "slug": "2026-03-27",
  "title": "Transformation Tech Weekly - March 27, 2026",
  "date": "2026-03-27",
  "sections": {
    "useful": [
      {
        "id": "optional-id",
        "title": "Title",
        "description": "Description",
        "reason": "Why useful/maybe useful/not useful",
        "tags": ["Tag1", "Tag2"],
        "url": "https://example.com"
      }
    ],
    "maybeUseful": [],
    "discarded": []
  }
}
```

Alternative keys are also accepted for compatibility:

- `usefulLinks`
- `maybeUsefulLinks`
- `discardedLinks`

## Analytics setup

Create `.env.local`:

```bash
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
```

Tracked events:

- `newsletter_link_click`
- `newsletter_link_feedback`

Event dimensions include newsletter slug, section, link ID/title, and vote.

## Deployment workflow

1. n8n generates JSON files from Excel.
2. JSON files are committed under `data/newsletters`.
3. Push to `main` triggers Vercel build.
4. Next.js statically generates updated pages for all newsletters.
